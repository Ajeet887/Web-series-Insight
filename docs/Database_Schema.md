# Database Schema Documentation

## 1. User Collection
Used for authentication and personalization.

| Field | Type | Description |
| :--- | :--- | :--- |
| `name` | String | Full name of the user |
| `email` | String | Unique identifier (lowercase) |
| `password` | String | Bcrypt hashed password |
| `preferences` | Object | Array of interests (genres, mood) |
| `createdAt` | Date | Timestamp of account creation |

---

## 2. Series Collection
Stores metadata and AI-generated insights.

| Field | Type | Description |
| :--- | :--- | :--- |
| `title` | String | Title of the web series |
| `description` | String | Plot summary |
| `genre` | Array | Categories (e.g., Sci-Fi, Drama) |
| `year` | Number | Release year |
| `platform` | String | Streaming service (Netflix, HBO, etc.) |
| `rating` | Number | Aggregated audience score |
| `cast` | Array | Lead actors |
| `director` | String | Showrunner/Director |

### Sub-Object: `insights`
Generated by the NLP Engine.
*   `whyLiked`: String array of positive themes.
*   `whyDisliked`: String array of negative themes.
*   `audienceType`: Targeted demographic (e.g., "Mystery lovers").
*   `contentDNA`: Object containing:
    *   `pacing` (Slow/Moderate/Fast)
    *   `complexity` (Low/Medium/High)
    *   `emotionalDepth` (Surface/Deep)

### Sub-Object: `reviews`
*   `userId`: Reference to User.
*   `text`: Raw review string.
*   `sentiment`: Object `{ positive, negative, neutral }`.
*   `topics`: Extracted keywords.

---

## 3. Relationships
*   **One-to-Many**: A User can have many reviews.
*   **Embedded**: Insights and Reviews are embedded within the Series document for faster read performance (Atomic Reads).
